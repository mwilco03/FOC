What Is Nmap?

  • Network Mapper — open-source security scanner since 1997
  • Created by Gordon "Fyodor" Lyon
  • Host discovery, port scanning, service/version detection, OS fingerprinting
  • 600+ NSE scripts for extended enumeration
  • Cross-platform: Linux, Windows, macOS

Common Use Cases

  • Asset inventory
  • Vulnerability assessment
  • Compliance auditing
  • Penetration testing
  • Incident response
  • Firewall validation

TCP Three-Way Handshake

  • SYN → Client initiates
  • SYN-ACK ← Server acknowledges
  • ACK → Connection established

Why It Matters

  • SYN scan -sS: never completes handshake (stealth)
  • Connect scan -sT: full handshake (logged)

CIDR Notation

  • /24 = 256 addresses
  • /16 = 65,536 addresses
  • /32 = single host
nmap -n -sL 10.10.1.0/24
# List targets without scanning

Host Discovery

nmap -sn 172.20.1.0/24    # Ping sweep — find live hosts
  • ARP on local subnet (can't be firewalled)
  • ICMP echo, TCP SYN/ACK on remote subnets
  • -Pn to skip discovery and scan anyway
🔬 Lab Challenge: How many hosts are alive on 172.20.1.0/24? Can you find the one that doesn't respond to ping?

open|filteredCan't determineCommon with UDP, use -sV

Service & OS Detection

nmap -sV -sC -O --traceroute 172.20.1.0/24

Version Detection (-sV)

  • Probes ports with protocol payloads
  • Identifies software name + version
  • --version-intensity 0-9

OS Fingerprinting (-O)

  • Sends TCP/UDP/ICMP probes
  • 6,000+ OS signatures
  • Requires open + closed port
🔬 Lab Challenge: What MySQL version is running? What's the SSH banner on port 2222?

Nmap Scripting Engine (NSE)

  • 600+ Lua scripts in /usr/share/nmap/scripts/
  • Categories: auth, vuln, discovery, exploit, safe
# Specific script
nmap --script smb-vuln-ms17-010 -p 445 target

# All vuln scripts on web ports
nmap --script vuln -p 80,443,8080 target

# Browse available scripts
ls /usr/share/nmap/scripts/ | grep http
🔬 Lab Challenge: Use ftp-anon to find anonymous FTP. Use redis-info to probe the unauth'd Redis server.

Blue Team: Detection

Detection Strategies

  • Connection-rate thresholds per source IP
  • Sequential port access patterns
  • SYN-only (half-open) connections
  • Unusual service probes (-sV payloads)
  • Correlate IDS + NetFlow data

Validation Exercise

# Simulate slow scan
nmap -T1 --max-rate 5 target

# Does your SIEM alert?
# If not, tune.

Test your own defenses. Scan yourself with T0-T5 and validate IDS catches all scan types.

Red Team: Evasion

# Decoy scan — mix your IP in with fakes
nmap -D RND:5,ME target

# Idle (zombie) scan — zero packets from you
nmap -sI zombie_host:80 target

Evasion Techniques

  • -f Fragment packets
  • --data-length <n> Pad packets
  • --source-port 53 Spoof as DNS
  • --randomize-hosts Avoid sequential patterns

⚠️ Authorization Required

These techniques are for authorized pentesting only. Always have written scope.

Legal & Ethical Considerations

⚠️ Disclaimer: Educational content, not legal advice. Always obtain written authorization.

Legal Landscape

  • No US law explicitly criminalizes port scanning
  • But unauthorized scanning → CFAA risk
  • ISPs may terminate accounts
  • International laws vary widely

Best Practices

  • Written authorization (SOW)
  • Define scope: IPs, ports, timing
  • Log everything you do
  • Report through proper channels
  • Respect scope boundaries
