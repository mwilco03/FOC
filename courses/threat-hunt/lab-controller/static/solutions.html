Availability & Observability (8 challenges)

Log Source Inventory — C (5 sources)

The Event Source Breakdown pie chart shows 5 index patterns: sysmon-*, zeek-*, winsec-*, applocker-*, winlogbeat-*.

Ingestion Health Check — C (applocker)

AppLocker only fires on blocked execution attempts. In this scenario, only one block event (invoice.exe) occurs, making it the lowest-volume source.

Coverage Gap — FLAG{3}

The attacker stopped Sysmon at 14:27 and restarted it at ~14:30. The Events Per Host timeline shows a 3-minute dropout for the victim host's Sysmon events.

Teaching point: A coverage gap IS the signal. Attackers disable logging tools — the absence of events is itself an indicator of compromise.
Priority Matrix — A (4, 1, 2, 3)

For USB-based kiosk attacks: AppLocker fires first (blocks initial attempt), Sysmon shows what actually executed, Zeek captures C2/exfil network activity, Windows Security provides supporting logon context.

Teaching point: Priority depends on the attack vector. For removable media threats on public terminals, execution-blocking controls (AppLocker) are your first line.
Dashboard Builder — FLAG{350}

Students build their own Kibana visualization. The peak ProcessCreate rate occurs during the discovery phase (~14:35) when the attacker runs ipconfig, net view, arp -a, and other commands in rapid succession.

Silent Host Detection — B (No — Winlogbeat still sends other event types)

Even when Sysmon stops, the host continues sending Windows Security, AppLocker, and other event channels via Winlogbeat. The Reporting Hosts metric stays at 1.

Teaching point: Multiple telemetry channels provide resilience. If an attacker kills one source, others continue. This is why defense-in-depth applies to logging too.
Observability Blind Spots — C (HTTPS payload content)

Zeek sees connection metadata, DNS queries, SMB file names, and unencrypted content. TLS-encrypted payloads (HTTPS) are opaque — Zeek logs the TLS handshake (JA3, server cert) but not the payload.

Teaching point: Know your blind spots. If C2 uses HTTPS, you need endpoint telemetry (Sysmon) or TLS inspection to see the payload. Network monitoring alone is insufficient.
Triage Priorities — B (Investigate why Sysmon stopped)

Other sources flowing = infrastructure is fine. One host's Sysmon dropping during an investigation means someone/something stopped it on that host. This is a high-priority indicator.

Teaching point: When triaging observability alerts, first determine scope (one host vs. all hosts). One host losing one source = host compromise. All hosts losing a source = infrastructure issue.
